Orden correcto de ejecución incremental

# 1. Verificar Fase 0 (ya corriste esto)
npm run test.unit -- --run --reporter=verbose

# 2. Al terminar Fase 1 → verificar acumulado
npm run test.unit -- --run --reporter=verbose

# 3. Al terminar Fase 2 → verificar acumulado
npm run test.unit -- --run --reporter=verbose

# 4. Al terminar Fase 3 → verificar acumulado
npm run test.unit -- --run --reporter=verbose

# 5. Al terminar Fase 4 → suite completa final
npm run test.unit -- --run --reporter=verbose
Regla de oro: Nunca avanzar a la siguiente fase si algún test de la fase anterior falla. Los tests son acumulativos — cada fase asume que la anterior funciona correctamente.

Troubleshooting rápido

ErrorCausa probableSolución
toBeInTheDocument is not a function setupTests no cargado o import incorrecto Verificar que setupFiles en vite.config.ts apunta a ./src/setupTests.ts
IDBFactory is not defined idb-keyval no mockeado El mock global en setupTests.ts lo resuelve. Verificar que no se importa idb-keyval directamente en el test.
useHistory outside Router AuthProvider sin MemoryRouter Usar renderWithProviders() que incluye el router por defecto
fetch not mocked Test hace fetch real Añadir vi.mocked(global.fetch).mockResolvedValueOnce(...) en el test
Test pasa solo pero falla en suite Contaminación de estado entre tests Añadir beforeEach(() => vi.clearAllMocks()) o limpiar el store IDB
